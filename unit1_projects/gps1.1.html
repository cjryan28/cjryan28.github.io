Pair Programming

What is Pair Programming?

Pair programming is a method of writing code where two people work on one project at the same time. Pair programming can take a few forms. The common ways to pair program are: driver/navigator and ping pong. Driver/navigator paring involves the navigator explaining what needs to be done, while the driver writes the code. In ping pong pairing, each of the pairs take turns on writing the code. Driver/navigator is most commonly used when one of the pair has a greater expertise.

Why Do Pair Programming?

Pair programming divides the labor of coding into two parts, which allows greater focus on each. The driver gets to focus on getting the code written effectively and accurately, while the navigator can analyze the direction of the project and copy-edit the syntax without having to split their attention with writing. By bringing two different skill sets and knowledge bases to bear on the same problem, each person helps mitigate the gaps in the other's experience and broadens the number of "tricks" available to the programmers.

How Do You Pair Program?

To be successful at pair programming, the most important quality for the pair to share is communication. It is essential that the pair discuss their plan, how to execute their plan, and any misconceptions along the way regarding the execution. Pair programming is a great way to learn, but only if the pairs make the most of it. It's important for each of the pairs to ask questions so that they can learn along the way.

Pair Programming at Dev Bootcamp

Dev Bootcamp brings together people with different skillsets and asks them to learn very quickly. The sharing of knowledge that is fundamental to pair programming accelerates that learning organically. Also, DBC puts enormous focus on developing empathy and teamwork in coding situations: pair programming is both a model and an experimental lab for those people skills.
